# Skills and Proficiency

## Source

* Book: SW5e Player's Handbook
* Pages: 208-209
* Original section: Proficiency Bonus, Expertise, Skills

## Summary

Proficiency represents training. Skills are common ways to apply ability checks, but the GM can pair a skill with a different ability when the described action calls for it.

## Key Rules

* Add proficiency bonus only when a character is proficient in the relevant skill, tool, save, or feature.
* Expertise or similar features can increase the proficiency contribution.
* A skill does not replace the ability; it modifies an ability check.
* Variant pairings let the GM call for unusual combinations, such as Strength (Intimidation).

## GM Use

Name checks as "Ability (Skill)" when useful. This helps players understand both the physical/mental approach and the trained technique.

## Common Rulings

* If the character's method changes, the ability can change.
* If a background, tool, or class feature is relevant, consider proficiency even when the default skill list is imperfect.
* Do not add proficiency twice unless a rule explicitly allows it.
